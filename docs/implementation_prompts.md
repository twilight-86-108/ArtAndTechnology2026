## 1. 本書の位置づけ

設計フェーズが完了し、実装フェーズに移行する。実装中はAIアシスタントを活用して開発速度を上げる方針（開発スケジュール5.2参照）。本書は、別チャットでAIアシスタントに支援を依頼する際のPromptテンプレートを提供する。

### 1.1 構成

| Prompt種別 | 目的 | 使用タイミング |
|----------|------|-------------|
| コアPrompt | プロジェクト全体の文脈を伝える | **全ての新規チャット冒頭** |
| フェーズ別Prompt | 現在のフェーズ固有の文脈を追加 | 該当フェーズの作業時 |
| トラブル別Prompt | 特定の問題解決に特化 | 該当の問題発生時 |

### 1.2 設計ドキュメントとの接続

本Promptは、以下の設計ドキュメントを前提とする。チャット開始時にこれらをアップロードして使う。

- 行動仕様書 v1.2
- 状態遷移図 v1.1
- 技術スタック選定書 v1.2.1
- データモデル設計 v1.1 + DataModelDefinitions.cs
- 視覚デザインガイド v1.1
- 開発スケジュール v2.0

## 2. Prompt運用方針

### 2.1 チャット分割の指針

| シナリオ | チャット分け方 |
|---------|------------|
| 1つの機能を集中実装 | 1機能 = 1チャット（例: 「セグメンテーション統合」） |
| 学習中の質問 | 1トピック = 1チャット（例: 「VFX Graphの使い方」） |
| 詰まったバグ調査 | 1バグ = 1チャット |
| 全体相談・進捗確認 | 週次レビュー = 1チャット |

### 2.2 コンテキストウィンドウへの配慮

- 長くなったチャットは「ここまでの要点をまとめて」と依頼し、新チャットに引き継ぐ
- ファイルアップロードは大きすぎないドキュメントに絞る（行動仕様書は45ページあるため、必要章のみ抜粋する場合もある）
- 1つのチャットで作業する時間は最大2〜3時間を目安にする

### 2.3 AIアシスタントの活用範囲

| 適している | 適していない |
|-----------|----------|
| Unity API/C#構文の質問 | 設計判断（自分で決めるべき） |
| エラーメッセージの解読 | 創作的な視覚表現の決定 |
| ボイラープレートコードの生成 | パラメータの最終調整（自分の感覚で） |
| アルゴリズムの実装支援 | 体験の「気持ちよさ」の評価 |
| デバッグの仮説立て | プロジェクトのコンセプト判断 |
| ドキュメント参照の高速化 | — |

## 3. コアPrompt（全チャット冒頭で使用）

### 3.1 標準コアPrompt

```markdown
# プロジェクト「境界生物 / Liminal Creature」実装支援

## プロジェクト概要

私は「境界生物 / Liminal Creature」というインタラクティブアート作品を一人で開発しています。
カメラに映った体験者のシルエットの輪郭線から生き物が生まれ、動きに反応し、関係性を築き、最終的に「通じ合い」の瞬間を迎える1分間の体験です。

## 基本情報

- **発表**: 2026年7月中旬（展示会形式）
- **開発期間**: 約14週間（420時間、週30時間）
- **私のスキル**: Unity初心者（数ヶ月以下）、PyTorchはビギナーレベル
- **開発体制**: 完全一人開発
- **使用技術**: Unity 6.3 LTS / URP / VFX Graph / Shader Graph / Inference Engine / MediaPipe Unity Plugin
- **OS**: Windows、ハードウェア: ラップトップ（GPU VRAM 8GB）

## 設計ドキュメント

このプロジェクトには6つの設計ドキュメントがあります。必要に応じてアップロードします。

1. 生き物行動仕様書 v1.2 - 行動ルール、反応、関係性
2. 状態遷移図 v1.1 - 6つの状態遷移（Mermaid）
3. 技術スタック選定書 v1.2.1 - 技術選定とMLパイプライン
4. データモデル設計 v1.1 + DataModelDefinitions.cs - データ構造
5. 視覚デザインガイド v1.1 - 色彩、形態、エフェクト
6. 開発スケジュール v2.0 - 14週間計画＋拡張

## 開発の進め方（重要）

私は以下の原則で開発しています:

- **3段階レベル設定**: Bronze（必達）→ Silver（目標）→ Gold（理想）
- **学習と実装を並行**: チュートリアルを最小限にし、本プロジェクトのコードを書きながら学ぶ
- **完璧を求めない**: 「動く」が最優先、「気持ちよく動く」は次点、「美しい」は余裕があれば
- **詰まったら30分で切替**: 30分考えて進まなければ別アプローチか別タスクへ

## 求める支援スタイル

1. **コードを書くときは「なぜそう書くか」も簡潔に説明してください** - 私が学習中のため
2. **複数の選択肢がある場合は推奨と理由を示してください** - ただし最終判断は私が行います
3. **設計ドキュメントと矛盾する提案は避けてください** - 矛盾する場合はその点を指摘してください
4. **Unity初心者前提で説明してください** - ただし過度に基礎的な説明は不要です
5. **できる限り Unity 6.3 LTS の最新APIに従ってください** - 古いAPIで提案する際は明示してください

## 現在の状況

[ここに現在の週、フェーズ、作業内容を書く。例: 「W3、フェーズ1の輪郭抽出実装中」]

## 今回の依頼

[具体的な依頼内容を書く]
```

## 4. フェーズ別追加Prompt

コアPromptに続けて使用する、各フェーズ固有の追加コンテキスト。

### 4.1 フェーズ1: 学習＋入力パイプライン（W1〜W3）

```markdown
## 現在のフェーズ: フェーズ1（学習＋入力パイプライン）

このフェーズの目標は、Unityの基本操作を身につけながらカメラ→セグメンテーション→輪郭抽出の入力パイプラインを構築することです。

### このフェーズで触る技術
- WebCamTexture でカメラ映像取得
- MediaPipe Unity Plugin (homuler) でセグメンテーション
- C#で Marching Squares による輪郭抽出
- Burst + Job System の基本

### 学習の方針
- チュートリアルは最大2日まで
- 本プロジェクトのコードを書きながら学ぶ
- AIアシスタントへの質問で学習速度を上げる

### このフェーズで触らない領域
- VFX Graph の高度な機能
- Shader Graph のカスタムHLSL
- 行動ロジック（フェーズ2以降）

### 完了条件
- カメラ映像が表示される
- セグメンテーションマスクが取得できる
- 輪郭点列が抽出できる
- 動きの速度が数値として取得できる
```

### 4.2 フェーズ2: プロトタイプ（W4〜W6）

```markdown
## 現在のフェーズ: フェーズ2（プロトタイプ・Bronze達成）

このフェーズの目標は Bronze レベルの達成です。
「パーティクルが輪郭から生まれ、動きに反応する」最小限の体験を完成させます。

### Bronze の定義
体験者がカメラの前に立つと、輪郭からパーティクルが生まれる。手を動かすと反応する（急→逃げる、ゆっくり→近づく）。人がいなくなるとパーティクルが消える。

### このフェーズで実装する機能要件ID
- 201（スポーン制御）
- 304（即時反応）
- 501（パーティクル描画）
- 506（背景描画）
- 801, 802, 805, 806（待機/開始/終了/リセット）

### このフェーズで使う技術
- VFX Graph（パーティクル描画、Additive合成）
- Shader Graph（シルエットエッジ）
- C#（行動ロジック、状態管理）

### このフェーズで実装しない機能
- 幼生・成体の形態変化（フェーズ3）
- 関係性の色変化（フェーズ3）
- 通じ合い演出（フェーズ3）
- 適応反応・同期検出（拡張フェーズ）

### 完了条件
- カメラの前に立つとパーティクルが生まれる
- 動きに反応する
- 人がいなくなると待機に戻る
- 30fps以上で動作
```

### 4.3 フェーズ3: コア体験構築（W7〜W10）

```markdown
## 現在のフェーズ: フェーズ3（コア体験構築・Silver/Gold達成）

このフェーズの目標は Silver レベル（時間が許せば Gold）の達成です。
成長システム＋関係性＋色変化、可能なら通じ合い演出まで実装します。

### Silver の定義
パーティクル→幼生→成体の成長。関係性に応じた寒色→暖色の色変化。

### Gold の定義（追加）
通じ合いトリガー判定と演出。保証メカニズム（48秒で強制発動）。

### 週ごとの目標
- W7: CreatureEntity、パーティクル集合→幼生化、幼生のVFX描画
- W8: 幼生→成体の成長、成体描画（単一Quad+SDF簡易版）、個性パラメータ
- W9: 関係性スコア、寒色→暖色変化、関係性状態遷移
- W10: 通じ合い判定＋演出、保証メカニズム、体験フェーズ制御

### 重要な設計判断
- パーティクルは VFX Graph 管理（CreatureEntity を生成しない）
- 幼生・成体は CreatureEntity として管理し、Object Pool で再利用
- 色補間は HSV 空間で行う（RGB直接補間ではくすむ）
- 全エフェクトは Additive 合成（カメラ映像に光を足す）

### 週ごとの判断ポイント
- W7終了時: 幼生化ができているか
- W8終了時: 成体が表示できているか
- W9終了時: 色変化ができているか

ダメなら次週はSilverを諦めてBronzeの品質向上に切替。
```

### 4.4 フェーズ4: 磨き込み＋安定化（W11〜W12）

## 5. トラブル別Prompt

特定の問題が発生した時に追加で使用する。

### 5.1 セグメンテーション精度が低い

```markdown
## トラブル: セグメンテーション精度の問題

現在使用しているモデル: [モデル名（例: MediaPipe SelfieSegmenter）]
症状: [具体的な症状（例: 輪郭がガタガタ、フリッカーが多い、人物が認識されない等）]

技術スタック選定書 20.2 のフォールバック戦略を念頭に、以下を判断したいです:
1. 現在のモデルで対処可能か（平滑化強度、信頼度閾値の調整等）
2. 別のモデルに切り替えるべきか（次の選択肢）
3. ファインチューニングを検討すべきか（時間的に可能か）

判断のための情報や、現在のモデルでの調整方法を提案してください。
```

### 5.2 パフォーマンス問題（FPS低下）

```markdown
## トラブル: パフォーマンス問題

現状: [現在のFPS、目標FPS]
プロファイル結果: [Profilerの主なボトルネック]

視覚デザインガイド 18.1 の描画負荷削減順序:
1. 環境パーティクル数削減
2. トレイル長削減
3. Bloom品質低下
4. ひれのNoise更新頻度削減
5. 生き物の描画数削減（最終手段）

この順序に沿って削減を試したいですが、プロファイル結果からどこから手をつけるべきか提案してください。
```

### 5.3 連続稼働でクラッシュ／メモリリーク

```markdown
## トラブル: 連続稼働での不安定性

症状: [N分でクラッシュ / N時間で動作が遅くなる等]
Profilerでの観察: [メモリ使用量の推移、特定オブジェクトの増加等]

技術スタック選定書 14.4 の自動復帰メカニズムは実装済み/未実装。
データモデル設計のObject Poolは適用済み/未適用。

考えられる原因と、調査・対策の優先順位を提案してください。
特にイベント購読の解除漏れ（EventBus.GetTotalSubscriberCount() が増え続ける）の可能性も含めて検討してください。
```

## 6. 使い方ガイド

### 6.1 新規チャット開始時の手順

```
1. 該当する設計ドキュメントをアップロード
   - 全フェーズ共通: 行動仕様書、データモデル設計
   - 視覚関連: 視覚デザインガイド を追加
   - 技術判断: 技術スタック選定書 を追加

2. コアPrompt（3.1標準版）をコピペ
   - 「現在の状況」と「今回の依頼」を埋める

3. 該当するフェーズ別Promptを追加
   - フェーズ1〜5から現在のフェーズを選ぶ

4. トラブルが発生している場合はトラブル別Promptも追加
```

### 6.2 Prompt使用例

#### 例1: フェーズ2でVFX Graphの実装相談

```
[コアPrompt 3.1]

現在の状況: W4、フェーズ2のパーティクル生成実装中

[フェーズ別Prompt 4.2]

[トラブル別Prompt 5.6]

実現したい視覚表現:
視覚デザインガイド7章のパーティクル仕様。輪郭上にスポーンし、Curl Noiseで漂う。
カメラ映像の上にAdditive合成で表示する。

現在試した方法:
VFX Graph で Output Particle Quad を使い、Initialize Particle で位置を指定するところまでできた。

問題点:
カメラ映像と合成すると、明るい部分でパーティクルが見えない。
Additive 合成にしているはずなのに沈んで見える。

Unity 6.3 LTS の VFX Graph での解決方法を提案してください。
```

#### 例2: トラブル解決

```
[コアPrompt 3.2 ミニマル版]

質問:
MediaPipe Unity Plugin を Unity 6.3 LTS で動かそうとしたが、以下のエラーが出る:

DllNotFoundException: mediapipe_c
  at (wrapper managed-to-native) ...

設定や導入手順で間違っていそうな点を教えてください。
```

### 6.3 Prompt運用のコツ

| コツ | 説明 |
|------|------|
| 設計ドキュメントは「必要な分だけ」 | 全部アップロードするとコンテキストを圧迫する |
| 「現在の状況」を毎回明示 | フェーズや週が変わると判断基準が変わる |
| 1チャット1テーマ | 複数の問題を混ぜると回答品質が落ちる |
| 「私の判断は最後」を明示 | AIの提案を鵜呑みにせず、必ず自分で判断する |
| 失敗例も共有 | 「これを試したがダメだった」を伝えると無駄な提案が減る |
| コードは抜粋して貼る | 全ファイルを貼らず、関連箇所のみ |

### 6.4 期待しすぎない

AIアシスタントは強力だが万能ではない。以下のことは自分で判断する:

- 体験の「気持ちよさ」の評価
- 視覚表現の最終的な美的判断
- スコープ調整の最終決定
- 設計ドキュメントの変更
- パラメータの最終的な調整

AIは「実装速度を上げる道具」として使い、「判断を委ねる相手」にはしない。
