## 2. ビジュアルコンセプト

### 2.1 コアイメージ: 精霊・妖精的

| 要素 | 方針 |
|------|------|
| 全体の印象 | 幻想的で柔らかい光。現実のカメラ映像の上に幻影が浮かび上がるような世界観 |
| 参照イメージ | 蛍の群れ、森の精霊、オーロラ、水面の光の反射 |
| 質感 | 半透明、発光体、輪郭が曖昧で空気に溶け込む |
| 動き | ふわふわと漂う、脈動する、風に吹かれるような揺らぎ |
| 避けるもの | 機械的・硬質な表現、明確な輪郭線、ピクセル的な表現 |

### 2.2 現実と幻想の境界

本作品のユニークな点は、背景がカメラのライブ映像であること。体験者は自分自身のリアルな姿を画面に見ながら、その輪郭から幻想的な生き物が生まれる。

この「現実と幻想の重なり」が視覚設計の軸であり、生き物は現実の映像に対して異質だが調和している、という微妙なバランスを保つ必要がある。

| 設計原則 | 説明 |
|---------|------|
| 生き物は発光する | 暗い場所でも明るい場所でもカメラ映像と視覚的に分離される |
| 生き物は半透明 | 完全に不透明だとカメラ映像を遮り「貼り付けた感」が出る |
| 環境パーティクルで空間を接続 | カメラ映像と生き物の間を環境パーティクルが埋め、世界観の統一感を出す |
| シルエットは淡く | 体験者の輪郭を強調しすぎず、生き物の「居場所」としてほのかに示す |

## 3. カラーパレット

### 3.1 基本パレット

関係性の進行に応じて寒色（冷たい・警戒）から暖色（温かい・信頼）へ変化する。

| 用途 | カラー名 | Hex | RGB | 使用場面 |
|------|---------|-----|-----|---------|
| 警戒ベース | Frost Blue | #A8D8EA | (168, 216, 234) | Wary状態の生き物の基本色 |
| 警戒アクセント | Ice White | #E8F4F8 | (232, 244, 248) | 警戒時のパーティクル |
| 興味ベース | Lavender | #C3B1E1 | (195, 177, 225) | Curious状態の基本色 |
| 興味アクセント | Soft Violet | #D8C5F0 | (216, 197, 240) | 興味時の発光色 |
| 親近ベース | Warm Peach | #F0C4A8 | (240, 196, 168) | Friendly状態の基本色 |
| 親近アクセント | Soft Gold | #F5DEB3 | (245, 222, 179) | 親近時の発光色 |
| 信頼ベース | Amber Glow | #FFB347 | (255, 179, 71) | Bonded状態の基本色 |
| 信頼アクセント | Warm White | #FFF5E6 | (255, 245, 230) | 通じ合い時の発光色 |
| 環境パーティクル | Dim Cyan | #4A7C8A | (74, 124, 138) | 背景に漂う微細パーティクル |
| シルエット | Ghost White | #F0F0FF | (240, 240, 255) | 体験者のシルエット輪郭 |

### 3.2 色遷移の補間

関係性スコア（0.0〜1.0）に応じたグラデーション補間。Shader GraphのLerpノードで実装。

```
score: 0.00 ─── 0.25 ─── 0.50 ─── 0.75 ─── 1.00
color: Frost    Lavender   Warm      Amber
       Blue                Peach     Glow

補間方式: HSV空間でのLerp（色相の回り込みを防ぐため）
  H: 200° (青) → 280° (紫) → 30° (橙) → 35° (琥珀)
  S: 0.28 → 0.28 → 0.30 → 0.72
  V: 0.92 → 0.88 → 0.94 → 1.00
```

> **設計判断**: RGB直接補間では中間色がくすむため、HSV空間で補間する。ただし色相の0°/360°境界を跨ぐルートを避けるため、紫(280°)を経由させる。

### 3.3 発光色

基本色に対する発光色は、基本色を明度+20%・彩度-30%したものを使用。Bloomポストプロセスで拡散する。

## 4. レイヤー構成

画面設計書6.4を視覚設計の観点から再定義。

### 4.1 レンダリング順序（奥→手前）

| 順序 | レイヤー | 内容 | 描画方式 |
|------|---------|------|---------|
| 0 | カメラ映像レイヤー | WebCamTextureをそのまま全画面表示 | 背景Quad or UI RawImage |
| 1 | 環境パーティクルレイヤー | 空間に漂う微細パーティクル | VFX Graph（加算合成） |
| 2 | 非対象者レイヤー | 非対象者のシルエット（薄く半透明） | Shader Graph（乗算暗転） |
| 3 | シルエットレイヤー | 対象者のシルエット輪郭（ほのかに発光） | Shader Graph（セグメンテーションマスクベース） |
| 4 | 生き物レイヤー | 生き物本体（パーティクル・幼生・成体） | VFX Graph（加算合成） |
| 5 | エフェクトレイヤー | 発光トレイル、通じ合いエフェクト | VFX Graph + Shader Graph（加算合成） |
| 6 | UIレイヤー | デバッグ情報（通常非表示） | UI Toolkit |

### 4.2 ブレンディング方針

| レイヤー | ブレンドモード | 理由 |
|---------|-------------|------|
| カメラ映像 | 不透明 | ベースレイヤー |
| 環境パーティクル | Additive（加算） | カメラ映像を暗くせず、光を追加する |
| 非対象者 | Multiply（乗算） | 映像を暗くして存在感を抑える |
| シルエット | Additive | ほのかな発光を載せる |
| 生き物 | Additive | 発光体として映像に光を追加 |
| エフェクト | Additive | 同上 |

> **重要**: 全ての生き物・エフェクトはAdditive合成。カメラ映像に対して光を「足す」ことで、明るい環境でも暗い環境でも自然に見える。Alphaブレンドだと明るい背景で生き物が沈む問題が起きるため、Additiveを基本とする。

## 5. 背景・環境演出

### 5.1 カメラ映像の処理

| 処理 | 方法 | 理由 |
|------|------|------|
| 表示 | WebCamTextureを背景Quadに描画 | 体験者が自分を認識するため |
| 色調補正 | なし（素のまま） | 現実感を維持する。フィルタをかけると不自然 |
| 明度抑制 | 要検討（暫定なし） | 明るすぎるとAdditiveの生き物が見えにくい。必要なら全体を70%に抑制 |

### 5.2 環境パーティクル

カメラ映像の上に漂わせ、「この空間には何かが存在する」という世界観を常に演出する。

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 色 | Dim Cyan (#4A7C8A)、透明度20〜40% | 控えめだがカメラ映像と区別可能 |
| サイズ | 1〜3px | 微細。主役を邪魔しない |
| 数 | 50〜100個 | パフォーマンス負荷を最小限に |
| 動き | 緩やかなPerlin Noiseによるドリフト | 風に漂うような自然な動き |
| ライフタイム | 5〜10秒（フェードイン/アウト） | 出現と消滅が目立たない |
| 分布 | 画面全体に均一 | 体験者の有無に関係なく常時表示 |

#### 体験中の環境パーティクル変化

| フェーズ | 変化 |
|---------|------|
| 待機 | 環境パーティクルのみ表示。穏やかな漂流 |
| 導入期 | 体験者の輪郭付近にパーティクル密度が上がる |
| 通じ合い期 | 環境パーティクルが生き物の色に一部染まる |
| 余韻期 | パーティクルが散って元の色に戻る |

### 5.3 待機画面の視覚演出（SCR-01）

> **v1.1追加**: STD-01のStandby状態で「待機中の生き物アニメーション」として表示されるもの。

体験者がいない状態では、カメラ映像の上に環境パーティクルと「待機中の生き物」が漂う。待機中の生き物は実際のCreatureEntityではなく、VFX Graphによるプリセットアニメーション。

| 要素 | 仕様 |
|------|------|
| 待機中の生き物 | 成体に似た発光体が2〜3体、画面内をゆっくり周遊。VFX Graphのプリセットアニメーション |
| 色 | Frost Blue〜Lavender の範囲でゆるやかに変化 | 
| 動き | 緩やかな8の字軌道。Perlin Noiseで揺らぎを加える |
| 反応 | なし（入力には反応しない。あくまで装飾） |
| 遷移 | 人物検出時に1秒かけてフェードアウトし、Guidance状態へ |
| 目的 | 「この画面は動いている（フリーズしていない）」ことを来場者に伝える。作品の世界観を予告する |

### 5.4 明るい背景への対策

> **v1.1追加**: Additive合成と明るいカメラ映像の相性問題への対処。

| 背景の明るさ | 問題 | 対策 |
|------------|------|------|
| 300〜500lux（標準） | 問題なし | 対策不要 |
| 500〜800lux（明るい） | HDR値2.0程度の加算が見えにくくなる | カメラ映像全体の明度を80%に抑制（Shader Graphで乗算） |
| 800lux以上（非常に明るい） | 加算がほぼ見えない | 明度を60%に抑制。生き物のHDR値を+1.0引き上げ |

明度抑制はキャリブレーション時に背景の平均輝度から自動判定する（技術スタック選定書14.3章参照）。管理画面（SCR-05）のプリセット「明るい」で手動設定も可能。

## 6. シルエット表現

### 6.1 対象者のシルエット

体験者自身がカメラ映像で見えているため、シルエットは「輪郭をほのかに強調する」程度に留める。生き物の「居場所」を示す役割。

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 表示方式 | セグメンテーションマスクのエッジを発光 | 内部は塗りつぶさない |
| エッジ幅 | 2〜4px | 太すぎると不自然 |
| 色 | Ghost White (#F0F0FF)、透明度30〜50% | 淡く光る |
| 脈動 | なし（静的） | 生き物との差別化。シルエットは「場」であり「生命」ではない |

#### Shader Graphでの実装方針

```
1. セグメンテーションマスクをテクスチャとして受け取る
2. マスクにガウシアンブラー（3×3 or 5×5）を適用してエッジのジャギーを軽減
3. ブラー後のマスクに対してSobelフィルタまたは膨張-収縮でエッジを抽出
4. エッジ部分にGhost White × 透明度をAdditiveで加算
5. マスク内部はカメラ映像をそのまま透過
```

> **v1.1追加**: セグメンテーションマスクのジャギー対策として、Sobelフィルタ適用前にガウシアンブラーを挟む。ブラーのカーネルサイズはパフォーマンスとエッジ品質のトレードオフで、プロトタイプで3×3と5×5を比較検証する。

### 6.2 シルエットと生き物の接続

生き物が輪郭上にいるとき、シルエットの対応箇所の発光がわずかに強まる。「ここに生き物がいる」という視覚的接続を提供する。

| 条件 | エッジの変化 |
|------|------------|
| 生き物がアンカーポイント付近にいる | エッジの該当部分の輝度が+50%、幅が+2px |
| 通じ合いトリガー時 | エッジ全体が一瞬（0.3秒）明るく脈動 |

## 7. パーティクル（導入期）

VFX Graphで制御。CreatureEntityとしては管理しない（データモデル設計5.3参照）。

### 7.1 外見

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 形状 | 円形のソフトパーティクル | ハードエッジなし |
| サイズ | 2〜5px | 微小。光の粒 |
| 色 | Frost Blue (#A8D8EA) → Ice White (#E8F4F8) | ライフタイムに応じて白へフェード |
| 透明度 | 60〜80% | 半透明 |
| 発光 | Bloom対応。HDR値 1.2〜1.5 | 柔らかく光る |
| テクスチャ | ガウシアンブラーの円形グラデーション | 中心が明るく端が透明 |

### 7.2 動き

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 生成位置 | 輪郭上のアンカーポイント（曲率ベース） | 行動仕様書5章準拠 |
| 初速 | 輪郭の法線方向に5〜15px/秒 | 輪郭から少し湧き出る |
| 減衰 | 初速 × 0.95/フレーム | すぐに止まり、漂い始める |
| 揺らぎ | Curl Noise (周波数0.5, 振幅3px) | ふわふわと漂う |
| 輪郭引力 | 微弱（輪郭から30px以内に留まる） | 離れすぎない |
| ライフタイム | 5〜10秒 | タイムアウトでフェードアウト |

### 7.3 集合（幼生化への遷移）

パーティクルが8〜12個集合すると幼生化する。集合の視覚演出:

| ステップ | 視覚表現 | 時間 |
|---------|---------|------|
| 集合開始 | 近隣パーティクルが互いに引き合い始める | 0〜1秒 |
| 集合中 | パーティクルが螺旋を描きながら1点に収束 | 1〜2秒 |
| 幼生化 | 光の閃光（0.1秒）→ 幼生体出現 | 一瞬 |

## 8. 幼生の視覚仕様

### 8.1 外見

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 形状 | 楕円形の光の塊。輪郭が曖昧 | 明確な形を持たない。アメーバ的 |
| サイズ | 15〜30px | パーティクルの3〜6倍 |
| 色 | 関係性スコアに応じた色（3.2参照） | 初期はFrost Blue |
| 透明度 | 40〜60% | かなり透ける |
| 発光 | HDR値 1.5〜2.0 | パーティクルより明るい |
| テクスチャ | 2層構造: 外殻（ぼやけた輪郭）+ コア（明るい中心） | |

### 8.2 脈動

幼生は常に脈動する。「生きている」感覚を与える最も重要なアニメーション。

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 脈動周期 | 1.5〜2.5秒（個体差あり） | energy パラメータで変調 |
| 脈動振幅 | スケール ±15% | 大きくなりすぎない |
| 脈動形状 | Easing: EaseInOutSine | 自然な膨張と収縮 |
| 発光脈動 | スケール脈動に同期。+20%の輝度変化 | 膨張時に少し明るく |

### 8.3 Shader Graphでの実装方針

```
VFX Graph (OutputParticleQuad) で描画:
  - Custom HLSL で楕円SDF（Signed Distance Field）を計算
  - SDFからソフトエッジを生成（smoothstep）
  - コア部分を明るく、外殻を透明にグラデーション
  - 脈動はVFX Graphのサイズアニメーションで制御
  - 色は CreatureStateBuffer からの関係性状態で決定
```

## 9. 成体の視覚仕様

### 9.1 外見

成体は幼生から成長した、より個性的な形態を持つ。ただし抽象的であり、具象的な生物の器官（目・口・手足等）は持たない。

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 形状 | 中心のコア + 複数の流動的な「ひれ」 | コアは楕円、ひれは動的に変形 |
| サイズ | 30〜80px | 個体差あり。energyが高いほど流線型 |
| 色 | 関係性スコアに応じた色（3.2参照） | 幼生から連続的に変化 |
| 透明度 | 30〜50%（コア）/ 10〜30%（ひれ） | コアが最も不透明 |
| 発光 | HDR値 2.0〜3.0 | 幼生より明るく、存在感がある |
| 軌跡 | 移動方向に淡いトレイルが残る | 14.2参照 |

#### 描画方式の選択

> **v1.1追加**: 成体の「コア＋ひれ」構造の描画アプローチ。

| 方式 | 説明 | メリット | デメリット |
|------|------|---------|----------|
| **A. 単一Quad + SDF（推奨）** | 1枚のQuadに対してShader内でSDF計算。コアとひれを1パスで描画 | 描画コール最小。VFX Graph1インスタンスで完結 | シェーダーが複雑になる。ひれの自由度に制約 |
| B. 複数Quad構成 | コア1枚 + ひれ2〜4枚で構成 | ひれの動きの自由度が高い | 描画コール増加。VFX Graph管理が複雑 |

方式Aを基本とし、SDF計算でひれの動きが表現不足な場合にのみ方式Bに切り替える。プロトタイプで検証。

### 9.2 個性による形態バリエーション

行動仕様書4.2章に対応。個性パラメータが外見に影響する。

| 個性傾向 | 形態特徴 | 実装方針 |
|---------|---------|---------|
| curiosity高 | 前方への伸長が大きい。「触角」的な光の突起 | SDFの前方パラメータを伸長 |
| timidity高 | コンパクト。ひれが短く体に密着 | 全体スケールを縮小、ひれ振幅を抑制 |
| energy高 | 流線型。ひれが長く、軌跡が長く残る | アスペクト比を横長に、トレイル長を増加 |
| sociability高 | コアの発光点が複数（2〜3点） | コアのSDF計算を複数中心点に拡張 |

### 9.3 ひれの動き

成体のひれは常に流動的に動き、静止しているときでも「生きている」印象を与える。

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| ひれの数 | 2〜4本（個体差） | sociabilityに応じて増減 |
| 動き | Perlin Noiseベースの波動 | 周期0.5〜1.0秒、振幅5〜15px |
| 移動時の変化 | 進行方向にひれが流れる | 速度ベクトルに応じてNoiseのオフセットを変化 |
| 逃避時の変化 | ひれが急速に収縮→体に密着 | 逃避開始0.1秒でひれ振幅を0に、0.5秒で回復 |

## 11. 関係性に応じた色調変化

機能要件ID 404に対応。

### 11.1 状態ごとの色と発光

| 関係性状態 | 基本色 | 発光色 | 発光強度(HDR) | 脈動速度 | 全体印象 |
|-----------|--------|--------|-------------|---------|---------|
| Wary | Frost Blue | Ice White | 1.5 | 遅い（2.5秒） | 冷たく、控えめに光る |
| Curious | Lavender | Soft Violet | 2.0 | やや速い（2.0秒） | 紫がかった柔らかい光 |
| Friendly | Warm Peach | Soft Gold | 2.5 | 速い（1.5秒） | 温かみのある光 |
| Bonded | Amber Glow | Warm White | 3.0 | 速い（1.2秒） | 強く温かい光。最も明るい |

### 11.2 色遷移アニメーション

状態遷移時の色変化は急激に変わらず、1〜2秒かけて滑らかに補間する。

```
遷移開始 → 1.0秒かけて新しい色へLerp
  Easing: EaseInOutQuad
  HSV空間で補間（3.2参照）
```

### 11.3 急な動きによる後退時の演出

関係性が後退する（例: Friendly → Curious）場合の視覚演出:

| 要素 | 演出 |
|------|------|
| 色 | 一瞬（0.2秒）白く点滅 → 後退先の色へ1秒かけて遷移 |
| サイズ | 一瞬収縮（×0.8） → 元に戻る |
| ひれ | 急速に収縮 → 1秒かけて回復 |

### 11.4 色覚多様性への配慮

> **v1.1追加**: カラーユニバーサルデザインの観点。

寒色→暖色の色遷移は色覚特性によっては差異が認識しにくい場合がある。色だけに依存せず、以下の複合的な変化で関係性状態を伝える:

| 視覚要素 | Wary | Curious | Friendly | Bonded |
|---------|------|---------|----------|--------|
| 色相 | 青 | 紫 | 橙 | 琥珀 |
| 輝度 | 低 (HDR 1.5) | 中 (2.0) | 高 (2.5) | 最高 (3.0) |
| サイズ | 小 | 中 | 中〜大 | 大 |
| 脈動速度 | 遅い (2.5秒) | やや速い (2.0秒) | 速い (1.5秒) | 最速 (1.2秒) |
| 動きの自由度 | 輪郭密着 | やや離れる | 自由に泳ぐ | 寄り添い |

> 色が認識できなくても、「暗くて小さくて動かない→明るくて大きくて活発」という輝度・サイズ・動きの変化で関係性の進行が伝わる設計。

## 18. パフォーマンスガイドライン

### 18.1 描画負荷の優先順位

パフォーマンスが不足した場合に品質を落とす順序:

| 順序 | 削減対象 | 方法 | 影響 |
|------|---------|------|------|
| 1（最初に削減） | 環境パーティクル数 | 100→50→25 | 雰囲気が薄くなるが体験には影響なし |
| 2 | トレイル長 | 30→15→5ポイント | 軌跡が短くなる |
| 3 | Bloom品質 | Scatter/Intensity を下げる | 発光が弱くなる |
| 4 | ひれのNoise解像度 | 更新頻度を半減 | ひれの動きがやや硬くなる |
| 5（最後に削減） | 生き物の描画数 | パーティクル上限を削減 | コア体験に影響 |

### 18.2 テクスチャとシェーダーの制約

| 項目 | 制約 |
|------|------|
| パーティクルテクスチャ | 32×32px以下。ガウシアンブラーのプロシージャル生成が望ましい |
| 生き物シェーダー | フラグメントシェーダーの命令数を最小限に。SDF計算はシンプルに保つ |
| ポストプロセス | Bloomのみ。追加のポストプロセスは使用しない |
| 描画コール | 生き物はVFX Graph一括描画。個別のGameObject化は避ける |

| 方向性表現の最適手法 | A〜Dの4手法（10章） | プロトタイプで全手法を実装し、ユーザーテスト | プロトタイプ検証後 |
| カメラ映像の明度抑制 | なし / 70%抑制 / 暗い部分のみ抑制 | 展示環境の照明条件で確認 | 展示環境テスト後 |

| 環境パーティクルの密度 | 50〜100個 | 体験者の視線追跡（あれば）or 感想ヒアリング | ユーザーテスト後 |

| 通じ合い演出の光の粒数 | 10〜30個 | パフォーマンスと演出効果のバランス | プロトタイプ検証後 |

| シルエットエッジのブラーサイズ | 3×3 / 5×5 ガウシアン | ジャギー軽減度とパフォーマンスを比較 | プロトタイプ検証後 |

### 20.2 視覚デザインの反復方針

視覚表現は数値だけでは最適解が見つからないため、以下の反復プロセスで調整する:

```
1. 本ガイドの数値でプロトタイプ実装
2. 自分自身で体験し、「気持ちよさ」を主観評価
3. パラメータを管理画面（SCR-05）からホットリロードで調整
4. 可能であれば第三者にテスト体験してもらい感想収集
5. 調整結果を settings.json に保存し、本ガイドにフィードバック
```
